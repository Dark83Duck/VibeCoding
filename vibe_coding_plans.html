바이브코딩이란?

오리엔테이션 & 캔바 탐색
Magic Media 캔바 기초
0–10분
1단계 · 도입
바이브코딩 세계로 초대
교사가 실시간으로 AI 이미지를 생성해 "코딩 없이 만든다"는 개념 전달 → 핵심 질문 3가지 제시 → 16차시 로드맵 공유
10–20분
2단계 · 개념
바이브코딩 3분 완성
기존 코딩 vs 바이브코딩 비교 → 좋은 프롬프트 vs 나쁜 프롬프트 실험 → 바이브(Vibe) = 스타일 + 분위기 + 색감 + 느낌
20–33분
3단계 · 실습
캔바 AI와 첫 만남
교육용 계정 로그인 → 인터페이스 탐색(만들기·앱·공유 버튼) → Magic Media로 "좋아하는 것 두 가지 합치기" 미션
33–38분
4단계 · 공유
내 첫 바이브 뽐내기
미니 갤러리 워크 → 프롬프트 한 줄 발표 → 교사가 프롬프트의 좋은 점 짚어 칭찬
38–40분
5단계 · 마무리
동아리 약속 & 다음 예고
AI 윤리 약속 3가지(초상권·정직·책임) → 2차시(Magic Media 심화) 예고 → 소감 한 마디
📌 핵심 교사 멘트
훅 멘트 (0분)"선생님이 방금 코드 한 줄도 안 짰어요. 그냥 하고 싶은 장면을 말로 써넣었을 뿐이에요. 여러분도 오늘부터 이걸 배웁니다."
프롬프트 막힐 때"AI가 네 마음을 못 읽은 것 같으면, 어떤 단어를 더 추가하거나 바꾸면 좋을까? 스타일을 바꿔볼까? 배경을 더 구체적으로 써볼까?"
🎯 오늘의 미션
실습 미션

자신이 가장 좋아하는 것 두 가지를 합쳐서 상상의 장면을 만들어 보세요.
공식: [누가/무엇이] + [어디서 무엇을 하는 중] + [스타일/느낌]
예) "우주비행사 고양이 + 달 위에서 라면을 먹고 있는 + 픽사 3D 스타일"

⚠️ 운영 팁
상황대응
로그인이 안 되는 학생미리 지정한 도우미 학생이 옆에서 도움, 교사는 전체 진행 유지
결과물이 마음에 안 드는 학생"프롬프트를 조금씩 바꾸며 재생성하는 것 자체가 핵심 학습"임을 안내
부적절한 이미지 시도즉시 개별 피드백, 분위기 망치지 않게 조용히

Magic Eraser & Magic Edit
Magic Eraser Magic Edit

2단계 · 개념
지우개 vs 편집 차이 이해
Magic Eraser: 원하지 않는 부분을 브러시로 칠하면 AI가 자연스럽게 지워 줌 / Magic Edit: 지운 자리에 새로운 오브젝트를 프롬프트로 채워 넣음 → 교사 실시간 시연 2회

33–38분
4단계 · 공유
Before & After 발표
수정 전·후 2장을 나란히 놓고 "무엇을 지우고 무엇을 넣었는지, 왜 그렇게 했는지" 발표
38–40분
5단계 · 마무리
편집 윤리 & 다음 예고
"실제 사람 사진을 무단으로 수정하면 안 되는 이유" 한 마디 → 4차시(Magic Write) 예고
⚠️ 운영 팁
상황대응
Eraser 브러시 크기 조절 몰라서 막히는 경우상단 슬라이더로 조절, 작은 브러시가 더 정교함
Edit 후 결과가 이상한 경우프롬프트를 더 구체적으로, 또는 Eraser 영역을 넓혀 재시도
실제 사람 사진 편집 시도조용히 개별 지도, AI 윤리 약속 재확인

Magic Write
0–7분

"AI가 쓴 글 vs 내가 쓴 글"

33–38분
4단계 · 공유
30초 앱 소개 발표

38–40분
5단계 · 마무리
AI 글쓰기 원칙 & 다음 예고

다듬기용 추가 프롬프트: "더 쉽고 친근한 말투로 바꿔줘" / "핵심만 남기고 짧게 요약해줘" / "30초 발표용 멘트로 바꿔줘"

내 앱 아이디어 기획하기

앱 기획 & 화면 흐름 설계
캔바 프레임 Magic Write
0–7분
1단계 · 도입
기획이란 무엇인가
4차시 소개글 회상 → 실제 앱 기획서 예시(간단한 버전) 보여 주기 → 오늘 목표: 나만의 앱 기획서 + 화면 흐름도 완성
7–15분
2단계 · 개념
앱 기획의 4가지 질문
① 누구를 위해? ② 무슨 문제를? ③ 어떻게 해결? ④ 화면은 몇 장? → 예시 앱(급식 메뉴 알림 앱)으로 4가지 질문 함께 작성
15–33분
3단계 · 실습
기획서 + 화면 흐름도
미션1: 4가지 질문 기획서 작성 (Magic Write 보조 활용) → 미션2: 종이에 화면 흐름도 손 스케치 (홈→메뉴1→메뉴2 형태) → 미션3: 캔바에 흐름도 슬라이드로 옮기기
33–38분
4단계 · 공유
모둠 기획 발표
모둠별 아이디어 한 줄 발표 → 친구들이 "써보고 싶은 앱"에 스티커(또는 손들기) → 가장 인기 있는 아이디어 칭찬
38–40분
5단계 · 마무리
기획서 저장 & 다음 예고
기획서 파일 이름 규칙 안내(예: "학번_앱기획_v1") → 6차시(UI 에셋 제작) 예고 → 오늘 기획서가 앞으로 16차시의 설계도가 됨을 강조

앱 이름: ________________ 한 줄 소개: ________________ 사용 대상: ________________ 해결하는 문제: ________________ 주요 기능 3가지: 1. ________________ 2. ________________ 3. ________________ 화면 구성 (몇 장): ________________장

나만의 UI 에셋 만들기

아이콘·버튼·배경 AI 제작 & 무드보드
Magic Media UI 요소
0–7분
1단계 · 도입
실제 앱 UI 뜯어보기
인기 앱(날씨, 유튜브, 카카오톡) 화면을 보며 "아이콘·버튼·배경이 각각 어디 있나요?" 찾기 → 오늘은 내 앱에 쓸 이 세 가지를 직접 AI로 만든다고 예고
7–15분
2단계 · 개념
UI 에셋 프롬프트 공식
아이콘: "심플한 앱 아이콘, 투명 배경, 플랫 디자인" / 버튼: "모바일 앱 UI 버튼, 둥근 모서리, 단색 배경" / 배경: "모바일 홈 화면 배경, 세로, 중앙이 단순한" → 교사 3가지 시연
15–33분
3단계 · 실습
에셋 3종 세트 + 무드보드
미션1: 내 앱 바이브에 맞는 아이콘 2개 생성 → 미션2: 버튼 이미지 1개 생성 → 미션3: 홈 화면 배경 1개 생성 → 미션4: 캔바 1장에 무드보드로 모아 정리
33–38분
4단계 · 공유
무드보드 갤러리
화면을 옆 친구에게 보여 주며 "내 앱의 바이브가 느껴지나요?" 확인 → 가장 통일감 있는 무드보드 2~3개 전체 공유
38–40분
5단계 · 마무리
에셋 폴더 정리 & 다음 예고
생성한 에셋을 캔바 폴더에 정리 저장 → 7차시에 이 에셋들로 실제 앱 화면을 조립한다고 예고
📋 에셋별 추천 프롬프트 앞부분
아이콘: "심플한 앱 아이콘, 투명 배경, 플랫 디자인 스타일, [색상], [주제]" 버튼: "모바일 앱 UI 버튼, 둥근 모서리, 단색 배경, [색상], [텍스트 없음]" 배경: "모바일 앱 홈 화면 배경, 세로 방향, 중앙이 비교적 단순한, [바이브 키워드]"
⚠️ 운영 팁
상황대응
투명 배경 아이콘이 안 나오는 경우프롬프트에 "white background"로 생성 후 캔바 배경 제거 기능 사용
버튼에 텍스트가 들어가서 어색한 경우"no text, icon only" 추가
바이브와 안 맞는 에셋이 나오는 경우2차시 바이브 카드를 옆에 열어 두고 스타일 키워드 그대로 복사
📦 차시 결과물
아이콘 2개 버튼 1개 배경 1개 무드보드 1장

바이브코딩 갤러리 전시회

발표 & 앱 체험 부스 운영
QR 공유 웹 게시
0–5분
1단계 · 준비
부스 세팅
각자 자리에 포스터 게시 + 태블릿/PC에 앱 프로토타입 열기 → QR 최종 스캔 확인 → 관람자용 스티커(별점) 배부
5–25분
2단계 · 전시
갤러리 오픈 (발표자·관람자 교대)
절반은 부스 운영(발표자), 절반은 관람(관람자) → 10분 후 역할 교대 → 관람자는 흥미로운 앱에 스티커 부착 + 한 마디 피드백 메모
25–33분
3단계 · 초청 관람
다른 반·선생님 초청 (선택)
사전에 초청한 다른 반 학생 또는 선생님들이 QR 스캔으로 앱 체험 → 발표자는 30초 설명 + Q&A
33–38분
4단계 · 시상
스티커 집계 & 칭찬 타임
스티커 수 집계 → 분야별 칭찬(가장 창의적인 앱·가장 실용적인 앱·가장 아름다운 디자인) → 교사가 전체 총평
38–40분
5단계 · 마무리
전시 사진 촬영 & 16차시 예고
전시 부스 단체 사진 촬영 → 16차시에 성장 포트폴리오와 수료식이 있다고 예고
📋 전시회 운영 체크리스트
  • 모든 학생의 포스터가 부스에 게시되었나요?
  • 태블릿/PC에서 앱 프로토타입이 열려 있나요?
  • QR 스캔 시 앱이 정상 연결되나요?
  • 관람자용 스티커와 피드백 메모지가 준비되었나요?
  • 역할 교대 시간(10분)이 타이머로 관리되고 있나요?

나의 바이브코딩 성장일지

회고 & 포트폴리오 완성 & 수료식
포트폴리오 Magic Write
0–7분
1단계 · 도입
1차시 첫 작품 vs 최종 작품
1차시 Magic Media 첫 이미지를 화면에 띄움 → 최종 앱 프로토타입과 나란히 비교 → "이 변화가 여러분의 성장입니다" 교사 총평
7–15분
2단계 · 개념
포트폴리오란 무엇인가
포트폴리오 = 내 성장의 증거 모음 → 오늘 만들 포트폴리오 3장 구조 안내: 1장(나의 바이브), 2장(대표 작품 3선), 3장(배운 것·다음 도전) → 완성 예시 시연
15–30분
3단계 · 실습
성장 포트폴리오 3장 완성
미션1: 1장 — 내 바이브 선언문 + 바이브 카드 삽입 → 미션2: 2장 — 16차시 중 가장 자랑스러운 작품 3개 선택·캡처·한 줄 설명 → 미션3: 3장 — "가장 많이 배운 것" Magic Write 초안 후 내 말로 수정
30–37분
4단계 · 수료식
바이브코딩 동아리 수료
교사가 학생 한 명씩 이름을 부르며 수료 한 마디 칭찬 → 디지털 수료증 (캔바 템플릿) 이름 써서 공유 → 전체 단체 박수
37–40분
5단계 · 클로징
바이브코더로 살아가기
"오늘 이후에도 말로 아이디어를 표현하고 AI와 협업하는 사람이 되세요" → 각자 소감 한 마디 → 동아리 공식 종료
📋 포트폴리오 3장 구성
장수제목포함 내용
1장나의 바이브바이브 카드 + "나는 __한 바이브코더입니다" 선언문
2장나의 대표 작품 3선작품 캡처 + 각 1문장 설명 (어떤 기능, 왜 자랑스러운지)
3장나의 성장 & 다음 도전"가장 많이 배운 것" + "다음에 도전하고 싶은 것"
📌 마지막 교사 멘트
수료식 클로징 멘트"여러분은 오늘부터 바이브코더입니다. 복잡한 코드가 없어도, 말로 아이디어를 표현하고 AI와 함께 만들 수 있다는 걸 이제 알죠? 이 능력은 앞으로 어디서든 쓸 수 있습니다. 계속 바이브하세요!"
